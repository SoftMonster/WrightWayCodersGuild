Employment Help for an Autistic Software Developer

Clear route into good technical work — without unnecessary friction, vague recruiter language, or performative hiring nonsense.

Who this page is for

This page is for an autistic software developer with:

  • 7 years of real software development experience
  • strong technical ability
  • evidence of useful delivered work
  • possible difficulty with standard interviews, vague recruiters, or overselling
Important: This is not an entry-level profile. It is a mid-level technical profile and should be treated that way.

Job Search Strategy

Do not use random mass applications. That wastes energy.

Bucket A — Best-fit roles

Apply when there is a 70–90% match.

  • Same stack
  • Same level
  • Mostly written communication
  • Clear responsibilities

Target: 5–10 strong applications

Bucket B — Adjacent roles

Apply when the work is very doable even if the title differs.

  • Software Developer → QA Automation
  • Backend → Platform / Integration
  • Full-Stack → Internal Tools

Target: 10–20 applications

Bucket C — Safe fallback

Useful if income needs restoring quickly.

  • Technical Support Engineer
  • Implementation Engineer
  • Systems Analyst
  • Support Developer

Proof-First Evidence Pack

1) One-page CV

  • Name
  • Title: Software Developer
  • Short summary
  • Skills
  • Work history
  • Selected projects
  • Links

2) Technical proof sheet

  • Strongest languages
  • Strongest frameworks
  • Tooling / infrastructure
  • Systems built
  • Examples of problem-solving
  • Examples of reliability
Example technical proof categories

Languages: Python, JavaScript, TypeScript, C#, Java
Frameworks: React, Node.js, Django, .NET, Spring
Infrastructure: Docker, CI/CD, AWS, Linux, Git
Work style: debugging, architecture, automation, refactoring, delivery

Best project evidence format
  • Problem: What needed fixing or building
  • Built: What was created or changed
  • Tech: Languages, frameworks, tools
  • Result: What improved
  • Proof: Link, screenshot, repo, or brief note

Interview Protection Plan

Useful adjustments can reduce noise without reducing standards.

Reasonable adjustments to request

  • Interview questions in advance
  • Written technical questions where possible
  • Clear agenda and timings
  • Reduced panel size
  • Extra processing time
  • Practical task instead of behavioural theatre

Correct positioning

Do not frame as:

  • “Looking for a chance”
  • “Passionate self-starter”
  • “Hard-working autistic candidate”

Frame as:

Software developer with 7 years of practical experience delivering useful technical work.

Copy-Paste Scripts

1) Message to specialist organisations

Hello, I am an autistic software developer with 7 years of experience. I am looking for software engineering or related technical opportunities. My strongest areas are: - [insert stack / strengths] I would like to know whether my profile may be suitable for current or upcoming roles. Thank you.

2) Interview adjustments request

Hello, I would like to request a few reasonable adjustments for the interview process. The most helpful adjustments would be: - clear interview structure in advance - written or practical technical questions where possible - explicit expectations for each stage Thank you.

3) Short personal introduction

Software developer with 7 years of experience building and improving software systems. Strong in structured problem-solving, debugging, implementation, and reliable delivery. Looking for a role with clear expectations, meaningful technical work, and a team that values competence.
